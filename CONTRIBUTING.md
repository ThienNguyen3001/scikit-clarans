# Contributing to scikit-clarans

Thank you for your interest in contributing to `scikit-clarans`! We welcome contributions from everyone.

## Types of Contributions

### Report Bugs

Report bugs at https://github.com/ThienNguyen3001/scikit-clarans/issues.

If you are reporting a bug, please include:

*   Your operating system name and version.
*   Any details about your local setup that might be helpful in troubleshooting.
*   Detailed steps to reproduce the bug.

### Fix Bugs or Implement Features

Look through the GitHub Issues for bugs or feature requests. Anything tagged with "good first issue" is a great place to start.

### Documentation

You can also help by improving the documentation, whether that's the README, docstrings in the code.

## Getting Started for Development

1.  **Fork** the repository on GitHub.
2.  **Clone** your fork locally:
    ```bash
    git clone https://github.com/your-username/scikit-clarans.git
    cd scikit-clarans
    ```
3.  **Install** the package in editable mode with development dependencies:
    ```bash
    pip install -e
    ```
4.  **Create a branch** for local development:
    ```bash
    git checkout -b name-of-your-bugfix-or-feature
    ```
5.  **Make your changes**.
6.  **Run tests** to ensure your changes don't break existing functionality:
    ```bash
    pytest
    ```
7.  **Commit** your changes and push your branch to GitHub:
    ```bash
    git add .
    git commit -m "Your detailed description of your changes."
    git push origin name-of-your-bugfix-or-feature
    ```
8.  **Submit a Pull Request** through the GitHub website.

## Pull Request Guidelines

Before you submit a pull request, check that it meets these guidelines:

1.  The pull request should include tests.
2.  If the pull request adds functionality, the docs should be updated.
3.  The pull request should work for Python 3.8+.

## Code Style

Please try to follow PEP 8 guidelines.
